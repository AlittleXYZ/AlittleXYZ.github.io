Nmap

Nmap介绍

Nmap，中文名”网络映射器”。一款开放源代码的网络探测和安全审核的工具。它能扫描大型网络和单个主机。

Nmap可以使用原始IP报文来发现网络上有哪些主机，这些主机提供什么服务，服务运行在什么操作系统，使用什么类型的报文过滤器或防火墙等等功能。

Nmap基本用法

简单的扫描

namp 192.168.126.131

该扫描方式可以针对IP或域名进行扫描，可以方便的发现目标端口的开放情况及主机在线情况。

扫描的结果可以很轻易的发现open端口，也可以发现相关的服务名称

Ping扫描(-sP)

nmap -sP 192.168.126.131/24

Ping扫描只进行Ping,显示在线的主机。该选项可以很轻易的获取目标信息而不会被轻易发现

无Ping扫描(-P0)

nmap -P0 192.168.126.131

无Ping扫描通常用于防火墙禁止Ping的情况下。可以穿透防火墙，避免被防火墙发现。

TCP STN Ping扫描(-PS)

nmap -PS -v 192.168.121.1

-PS选项发送一个设置了SYN标志位的空TCP报文，默认端口80，可以指定一个以逗号分割的端口列表(-PS22,23,25)

默认的Ping扫描使用TCP ACK和ICMP Echo请求对目标进行是否存活的响应。当目标主机防火墙组织这些请求时，可以使用TCP SYN Ping扫描进行对目标主机存活的判断

我们也可以whiffing一个端口或范围来避免扫描结果的不准确

nmap -PS80,100-200 -v 192.168.121.1

TCP ACK Ping扫描(-PA)

nmap -PA -v 192.168.121.1

与TCP SYN Ping扫描的区别就是设置TCP的标识为是ACK而不是SYN，这种方式可以探测阻止SYN包或ICMP Echo请求的主机

UDP Ping扫描(-PU)

nmap -PU -v 192.168.121.1

-PU发送一个空的UDP报文到指定端口，不指定则默认端口40125。

使用UDP Ping扫描时候Nmap会发送一个空的UDP包到目标主机，如果目标主机响应会返回一个ICMP端口不可达错误，如果目标主机不是存活状态则会返回各种ICMP错误信息

ICMP Ping Types扫描(-PE;-PP;-PM)

ICMP(Internet控制报文协议)，是TCP／IP协议族的一个子协议，用于在IP主机、路由器之间传递控制消息(指网络通不通、主机是否可达、路由是否可用等网络本身的消息)。

-PE

nmap -PE -v 192.168.121.1

-PE是通过向目标发送ICMP Echo数据包来探测目标主机是否在线，因为许多防火墙会禁止这些报文，所以仅仅ICMP扫描对于互联网上的目标通常是不够的

-PP

nmap -pp -v 163.com

-PP选项是ICMP时间戳Ping扫描，虽然大多数的防火墙配置不允许ICMP Echo请求，但配置不当可能恢复ICMP时间戳请求，所以可以使用ICMP时间戳来确定目标主机是否存活。

-PM

nmap -PM -v 192.168.121.1

-PM选项可以进行ICMP地址掩码Ping扫描。这种扫描会试图用备选的ICMP等级Ping制定主机，通常有不错的穿透防火墙的效果

ARP Ping扫描(-PR)

nmap -PR 192.168.126.131

-PR选项通常在扫描局域网时使用。ARP Ping扫描是Nmap对目标进行一个ARP Ping的过程。在本地局域网中防火墙不会禁止ARP请求，这就使得其他Ping扫描都更加高效，在内网中使用ARP Ping是非常有效的。

扫描列表(-sL)

nmap -sL 192.168.121.131/24

列表扫描是主机发现的退化形式，它仅仅列出指定网络上的每台主机，不发送任何报文到目标主机

禁止反向域名解析(-n)

nmap -n -sL 124.172.156.75/24

-n意为禁止解析域名，使用该选项的时候Nmap永远不对目标IP地址做反响域名解析

反向域名解析(-R)

nmap -R -sL /*.172.156.75/24

使用系统域名解析器(–system-dns)

nmap –system-dns 192.168.126.2 192.168.126.131

扫描一个IPV6地址(-6)

nmap -6 fe80::20c:29ff:fee0:2e76

路由跟踪(–traceroute)

nmap –traceroute -v www.163.com

使用路由跟踪功能可以帮助用户了解网络的同行情况，通过此选项可以轻松查出从本地计算机到目标之间所经过的网络节点，并看到通过各个节点的时间

SCTP INIT Ping扫描

SCTP(流控制传输协议)是IETF在2000年定义的一个传输层协议。SCTP可以看作是TCP协议的改进。

nmap -PY -v 198.168.121.1

端口扫描

时序选择(-T)

Nmap使用-T(0-5)启用时序选择

  1. -T0：非常慢的扫描，用于IDS逃避
  2. -T1：缓慢的扫描，用于IDS逃避
  3. -T2：降低速度以降低对带宽的消耗，一般不常用
  4. -T3：默认，根据目标的反应自动调整时间
  5. -T4：快速扫描，常用扫描方式，需要在很好的网络环境下进行扫描
  6. -T5：极速扫描，牺牲准确度来提升速度

nmap -T0 192.168.126.131

常用扫描方式(-p|-F)

-p

该选项指定一个你想要扫描的端口号，可以指定一个也可以指定一个范围

nmap -p 445 192.168.126.131

-F

该选项可以快速的扫描端口，只会扫描有限的端口

nmap -F 192.168.126.131

-r

默认情况Nmap是进行随机顺序扫描的，使用该选项可以进行排序

–top-ports

nmap –top-ports 100 192.168.126.131

上述结果为扫描一百个开放率最高的端口

–port-ratio

使用该选项会扫描指定一定概率以上的端口，这里的参数作为频率使用

TCP SYN扫描(-sS)

SYN扫描方式是相对比较隐蔽的扫描方式，很难被防火墙或管理员发现，因为它不会进行TCP的链接。这是一种高效有用的方式

它常常被称为半开放扫描。

nmap -sS 192.168.126.131

TCP连接扫描(-sT)

它用于SYN扫描不能使用的时候，Nmap会在每个端口完成三次握手，所以TCP连接扫描是端口中最基础、最稳定的扫描方式

nmap -sT 192.168.126.131

UDP扫描(-sU)

UDP扫描是十分慢的，很多管理员会忽略这些端口，使用UDP扫描或许可以发现更多的被黑客利用的端口。

UDP扫描发送空的UDP报文到目标端口，如果返回ICMP端口不可到达错误就可以认定端口是关闭的，其他的就可以认定是被过滤的，如果被响应了则判断目标端口是开放状态

nmap -sU -p 80-500 192.168.126.131

由于UDP扫描是相当慢的，所以使用-p选项指定需要扫描的端口

隐蔽扫描(-sN;-sF;-sX)

-sN

Null扫描，通过发送非常规的TCP通信数据包对计算机进行探测，很多情况Null扫描与Xmas恰好相反，因为Null不会标记任何数据包。若目标主机的相应端口是关闭的，会响应一个RST数据包，若目标端口开放则不会响应任何信息

-sF

FIN扫描，当我们使用TCP SYN扫描可能会被目标主机的防火墙发现，会阻止SYN数据包，当使用TCP FIN就有很好的穿透效果。

-sX

Xmas扫描。

这些扫描方式会躲过一些无状态防火墙的过滤。比SYN扫描，UDP扫描会有更好的效果，因为这种方式比SYN扫描还要隐蔽。如果扫描不出结果或被过滤的情况下，可以使用该扫描方式。

nmap -sN 192.128.126.131

TCP ACK扫描(-sA)

它有一个非常致命的缺点，它不能确定端口是否开放的还是关闭的

nmap -sA -v 192.168.121.1

TCP窗口扫描(-sW)

窗口扫描方式与ACK扫描方式的原理几乎一样

nmap -sW -v -F 192.168.121.1

自定义TCP扫描(–scanflags)

使用该选项可以启用自定义TCP扫描，属于高级用法

–scanflags使用字符名，只要是URG;ACK;PSH;RST;SYN;FIN任何组合就可以。标识位的顺序不重要

nmap -sT –scanflags SYNURG 192.168.126.131

空闲扫描(-sI)

空闲扫描是Nmap高级用法

允许进行端口完全欺骗扫描。可以是攻击者能够不使用自己的IP向目标主机发送数据包，可以利用不活跃的僵尸主机反弹给攻击者一个旁通信道，进行端口扫描。

nmap -sI www.0day.co:80 192.168.126.131

这是利用僵尸主机位www.0day.co的主机对102.168.126.131进行空闲扫描

IP协议扫描(-sO)

该选项会确定目标端口的协议类型，IP协议扫描可以帮助用户确定主机支持那些IP协议，例如TCP、ICMP、IGMP。

nmap -sO -T4 102.168.126.131
